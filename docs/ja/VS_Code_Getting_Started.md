# VS Codeを始めよう

コードを書きたいけど、メモ帳やテキストエディットでは物足りない。シンタックスハイライト、オートコンプリート、内蔵ツールでより速くコーディングできるものが必要です。**VS Codeはコード用のスマートノートブックのようなものです** - あなたが書いていることを理解し、途中で助けてくれます。[Visual Studio Code](https://code.visualstudio.com)は無料で、どのOSでも動作し、何百万もの開発者に使用されています。

## 主要な概念

- **エディター** - コードを書くメインエリア
- **エクスプローラー** - ファイルとフォルダを表示するサイドバー
- **拡張機能** - VS Codeに新しい機能を追加するアドオン
- **統合ターミナル** - VS Codeに内蔵されたコマンドライン

## 必要なもの

- Windows、macOS、またはLinuxコンピュータ
- インターネット接続
- 約500 MBのディスク空き容量
- 20〜25分

## ステップ1：VS Codeをダウンロードしてインストール

- [code.visualstudio.com](https://code.visualstudio.com)にアクセス
- **ダウンロード**ボタンをクリック（OSを自動検出します）
- インストーラーを実行：
   - **Windows：** `.exe`をダブルクリックし、プロンプトに従って**次へ**をクリック
   - **Mac：** `.dmg`を開き、VS Codeを**アプリケーション**にドラッグ
   - **Linux：** パッケージマネージャーの指示に従う
- VS Codeを起動

始めるオプションを含むウェルカムタブが表示されます。

## ステップ2：インターフェースのツアー

VS Codeには5つの主要なエリアがあります：

- **アクティビティバー**（左端） - エクスプローラー、検索、Git、拡張機能などのアイコン
- **サイドバー** - 選択したアクティビティのコンテンツを表示（ファイル、検索結果）
- **エディター**（中央） - コードを書く場所
- **パネル**（下部） - ターミナル、問題、出力
- **ステータスバー**（最下部） - ファイルとプロジェクトに関する情報

**エクスプローラー**アイコン（アクティビティバーの上部）をクリックしてファイルブラウザを表示。

## ステップ3：フォルダを開いて探索

- **ファイル** → **フォルダーを開く**をクリック
- コンピュータ上の既存のフォルダに移動（例：ドキュメント）
- **開く**（または**フォルダーの選択**）をクリック
- 「このフォルダー内のファイルの作成者を信頼しますか？」と聞かれたら、**はい、作成者を信頼します**をクリック

エクスプローラーサイドバーにフォルダのファイルが表示されます：

- フォルダをクリックして展開
- 任意のファイルをクリックしてエディターで開く
- アクティビティバーの**検索**アイコン（虫眼鏡）をクリックして全ファイルを検索

## ステップ4：Markdownファイルを作成

- エクスプローラーで**新しいファイル**アイコン（+のついたページ）をクリック
- `README.md`と名前を付ける
- 以下の内容を追加：

```
# マイプロジェクト

これはVS Codeを学ぶための**デモプロジェクト**です。

## 機能
- 編集が簡単
- Markdownフォーマット
- ライブプレビュー

## 次のステップ
1. コンテンツを追加
2. 他のファイルタイプを試す
3. 拡張機能を探索
```
- **ファイル** → **保存**をクリックして保存

Markdownは`#`を見出しに、`**`を太字に、`-`をリストに使うシンプルなテキスト形式です。ドキュメンテーションやChatGPT、ClaudeなどのLLMとのコミュニケーションに広く使用されています。

## ステップ5：Markdown Preview拡張機能をインストール

- アクティビティバーの**拡張機能**アイコンをクリック（四角形のアイコン）
- 検索ボックスに`Markdown Preview Enhanced`と入力
- 結果から**Markdown Preview Enhanced**を見つける
- **インストール**をクリック

## ステップ6：Markdownファイルをプレビュー

- `README.md`がまだ開いていなければ開く
- エディター内で右クリックし、**Markdown Preview Enhanced: Open Preview to the Side**を選択

フォーマットされたMarkdownを表示するプレビューパネルが開きます - 左で編集し、右でリアルタイムに変更を確認！

## ステップ7：統合ターミナルを使用

- **ターミナル** → **新しいターミナル**をクリック（またはバッククォートで`` Ctrl+` ``を押す）
- これらのコマンドを試す：

**ファイルをリスト：**
```
ls
```
（Windowsコマンドプロンプトでは`dir`を使用）

**作業ディレクトリを表示：**
```
pwd
```
（Windowsコマンドプロンプトでは`cd`を使用）

**新しいフォルダを作成：**
```
mkdir notes
```

エクスプローラーを確認 - `notes`フォルダが表示されます！ターミナルはプロジェクトフォルダで実行されるので、コマンドは直接プロジェクトに影響します。

## ステップ8：VS CodeでAIエージェントを使用（オプション）

VS Codeには[GitHub Copilot Chat](https://code.visualstudio.com/docs/copilot/chat/getting-started-chat)が含まれています。これはコードの説明、作成、デバッグができるAIアシスタントです。

- `README.md`ファイル（またはプロジェクト内の他のファイル）を開く
- タイトルバーの**Chat** → **Open Chat**をクリック（またはWindows/Linuxで`Ctrl+Alt+I`、Macで`Ctrl+Cmd+I`を押す）
- プロンプトが表示されたら、**GitHubアカウント**でサインイン（無料プランあり）
- 開いたチャットパネルで「このファイルを説明して」と入力
- **Enter**を押す

GitHub Copilotがファイルを分析し、何をするか説明します。新しいコードを書いたり、バグを修正したり、「さらに機能を追加するにはどうすればいい？」などの質問に答えてもらえます。

**ヒント：** [インライン編集](https://code.visualstudio.com/docs/copilot/copilot-chat)の場合、任意のファイルでコードをハイライトし、`Ctrl+I`（Windows/Linux）または`Cmd+I`（Mac）を押すと、Copilotにその部分だけを修正、変更、または説明してもらえます。

## プロジェクトを再度開く方法

- スタートメニュー（Windows）、Spotlight（Mac）、またはアプリケーション（Linux）からVS Codeを開く
- **ファイル** → **最近使用した項目を開く** → フォルダを選択
- または**ファイル** → **フォルダーを開く**でナビゲート

## トラブルシューティング

- **プレビューが表示されない：** Markdown Preview Enhanced拡張機能がインストールされていて、`.md`ファイルが開いていることを確認
- **ターミナルが間違ったディレクトリを表示：** ターミナルパネルのゴミ箱アイコンをクリックし、**ターミナル** → **新しいターミナル**をクリック
- **チャットメニューが表示されない：** GitHub Copilot Chatのインストールが必要かもしれません - **拡張機能**アイコンをクリックし、「GitHub Copilot Chat」を検索してインストール
- **拡張機能が動作しない：** **表示** → **コマンドパレット**をクリックし、「reload window」と入力し、**Developer: Reload Window**を選択

## 完全なワークフロー

1. VS Codeでフォルダを開く
2. ファイルを作成/編集
3. 拡張機能でプレビュー
4. ターミナルでコマンド実行
5. AIでコードを理解・改善
6. 保存して繰り返す

## 次のステップ

- **他のファイルタイプを試す：** `.html`、`.css`、`.js`、または`.py`ファイルを作成してVS Codeのシンタックスハイライトを確認
- **AI機能を探索：** AIにコードを書いてもらったり、バグを修正してもらったり、プロジェクトの改善を提案してもらう
- **さらに拡張機能をインストール：** 自動フォーマット用の「Prettier」やGit機能用の「GitLens」を試す
- **ショートカットを学ぶ：** **ヘルプ** → **キーボードショートカット参照**をクリックしてワークフローを高速化
