# Mac版Claude Codeでローカルバージョン管理

AIの支援を受けてコーディングしています。AIがファイルを変更します。変更がうまくいくこともあれば、そうでないこともあります。**バージョン管理はプロジェクト全体の「元に戻す」ボタンのようなものです。** スナップショット（「コミット」と呼ばれる）を保存するたびに、いつでも戻れる復元ポイントを作成します。さらに、これをすべてコンピュータ上で行うことができ、Claude Codeが代わりにやってくれます。

## 主要な概念

- **ターミナル** - コマンドを実行するためのMac内蔵のコマンドラインインターフェース
- **Git** - コンピュータ上のファイルへのすべての変更を追跡し、いつでも戻れる復元ポイントを作成
- **コミット** - 変更内容の説明とともに、特定の時点でのプロジェクトのスナップショット
- **Claude Code** - シンプルなリクエストでコードを書き、バグを修正し、Git操作を処理するAIコーディングアシスタント

## 必要なもの

- [MacへのClaude Codeのインストール](./Install_Claude_Code_MacOS)を完了
- 20分

## ステップ1：ターミナルを開く

以下のいずれかの方法を選択：

- **Spotlight：** `Command (⌘) + Space`を押し、`ターミナル`と入力し、Enterを押す
- **Finder：** **アプリケーション** > **ユーティリティ** > **ターミナル**を開く
- **Launchpad：** Dockの**Launchpad**をクリックし、`ターミナル`を検索

`$`または`%`で終わるコマンドプロンプトが表示されます。

## ステップ2：Gitをインストール

Macには通常Gitがプリインストールされています。確認しましょう：

- このコマンドを入力してEnterを押す：
  ```
  git --version
  ```

**バージョン番号が表示された場合**（`git version 2.39.0`のような）、Gitはすでにインストールされています—ステップ3に進んでください。

**「command not found」と表示された場合**、または開発者ツールをインストールするポップアップが表示された場合：
- ポップアップの**インストール**をクリック、または以下を実行：
  ```
  xcode-select --install
  ```
- インストールが完了するまで待つ（1〜5分）
- Gitがインストールされたことを確認：
  ```
  git --version
  ```

`git version 2.39.0`のような表示が見えるはずです。

## ステップ3：Gitに身元を設定

Gitはコミットメッセージのためにあなたが誰かを知る必要があります。

- 名前とメールアドレスを設定（偽でも可）：
  ```
  git config --global user.name "Your Name"
  git config --global user.email "your.email@example.com"
  ```

名前とメールアドレスを使用すると、複数人が作業する場合に誰が変更を加えたかを識別するのに役立ちます。

## ステップ4：書類フォルダに移動

- 書類フォルダに移動：
  ```
  cd ~/Documents
  ```
- 正しい場所にいることを確認：
  ```
  pwd
  ```

`/Users/YOUR_USERNAME/Documents`と表示されるはずです。

## ステップ5：プロジェクトフォルダを作成

- `timer`という名前のフォルダを作成：
  ```
  mkdir timer
  ```
- フォルダに移動：
  ```
  cd timer
  ```

これがプロジェクトの場所になります。

## ステップ6：Claude Codeを起動

- Claude Codeを起動：
  ```
  claude
  ```

Claude Codeが起動し、リクエストを待ちます。

## ステップ7：ClaudeにGitを初期化させる

- このリクエストを入力：
  ```
  変更の追跡を開始して
  ```

Claudeがフォルダ内にGitリポジトリを初期化します（2〜5秒かかります）。これでバージョン管理ができます！

## ステップ8：タイマーアプリを作成

- Claude Codeで以下を入力：
  ```
  シンプルなカウントダウンタイマーアプリをtimer.htmlという単一ファイルで作成してください。
  必要な機能：
  - 分を設定する入力フィールド
  - 開始と停止ボタン
  - MM:SS形式で残り時間を表示
  ```

Claudeが`timer.html`を作成します（10〜30秒かかります）。CSSとJavaScriptコードが含まれます。

## ステップ9：タイマーをテスト

- Finderを開く
- **書類** > **timer**に移動
- `timer.html`をダブルクリックしてブラウザで開く
- タイマーを試す：
  - 入力フィールドに`1`と入力
  - **開始**をクリック
  - カウントダウンを見る

**何か壊れている場合：** Claude Codeでエラーを説明：`このエラーが出ています：[何が起こったか説明]。修正できますか？`

## ステップ10：Claudeにコミットを依頼

- Claude Codeで以下を入力：
  ```
  この変更を保存して
  ```

Claudeが以下を行います：
- どのファイルが変更されたか確認
- 説明的なコミットメッセージを書く
- コミットを作成（5〜10秒かかります）

最初の保存ポイントを作成しました！いつでもこの動作するバージョンに戻れます。

## ステップ11：プリセットボタンを追加

- Claude Codeで以下を入力：
  ```
  上部に2つのボタンを追加してください。クリックすると自動的に1分と5分のタイマーが開始するようにしてください。
  ```
- ブラウザタブを更新（または`Command (⌘) + R`を押す）
- テスト：**5 min**ボタンをクリック
- 動作したら、変更をコミット：
  ```
  この変更を保存して
  ```

2つ目の保存ポイントを作成しました。このバージョンには2つのボタンが動作しています。


## ステップ12：別のボタンを追加

- Claude Codeで以下を入力：
  ```
  15分ボタンを追加してください。
  ```
- ブラウザタブを更新（`Command (⌘) + R`を押す）
- テスト：**15 min**ボタンをクリック

**このチュートリアルでは：** 15分ボタンが正しく動作しないふりをしてください。まだコミットしないでください。悪い変更を破棄する練習をします。

## ステップ13：変更を破棄

AIのコードが動作せず、最後の保存ポイントからやり直す必要がある場合があります。

- Claude Codeで以下を入力：
  ```
  この変更を破棄して
  ```
- Claudeが確認を求めます
- `yes`と入力してEnterを押す
- ブラウザを更新—15分ボタンが消えます

Claudeが気に入らない新しい変更を捨てます。タイマーは再び1分と5分のボタンだけで動作します！

## ステップ14：サウンド通知を追加

- Claude Codeで以下を入力：
  ```
  時間になったらサウンド通知を追加してください。
  ```
- Claudeが完了したらブラウザを更新してテスト（タイマーを0.1分に設定）
- 動作したら、変更をコミット：
  ```
  この変更を保存して
  ```

## ステップ15：スヌーズボタンを追加

- Claude Codeで以下を入力：
  ```
  スヌーズボタンをクリックするまでサウンドが続くようにしてください。
  ```
- Claudeが完了したらブラウザを更新してテスト（タイマーを0.1分に設定）
- 動作したら、変更をコミット：
  ```
  この変更を保存して
  ```

## ステップ16：コミット履歴を表示

- Claude Codeで以下を入力：
  ```
  変更履歴を表示して
  ```

Claudeが読みやすい形式でコミットを表示します。以下が見えるはずです：
- 最初のタイマーアプリのコミット
- プリセットボタン（1分と5分）のコミット
- サウンド通知のコミット
- スヌーズボタン

15分ボタンの試行がないことに注目してください—破棄しました！

## ステップ17：コードを調べる

- アプリを表示しているブラウザで、右クリックして**ページのソースを表示**を選択（または`Option (⌥) + Command (⌘) + U`を押す）
- ソースコードを見ることができます
- Claude Codeで質問：
  ```
  このコードを説明して。大まかな概要だけで。
  ```

## 完全なワークフロー

- Claudeに変更を依頼
- テスト
- 動作したら → Claudeにコミットを依頼
- 失敗したら → Claudeに修正を依頼
- 修正できない → 変更を破棄して再試行
- 繰り返す

いつでも任意のコミットに戻れます。恐れずに破棄してください—動作するコードだけをコミットしましょう！

## 次のステップ

タイマーにさらに機能を追加してみましょう：

- **15分ボタン：** `動作する15分プリセットボタンを追加`（破棄したものをやり直し！）
- **一時停止ボタン：** `タイマーの状態を切り替える一時停止/再開ボタンを追加`
- **より良いスタイル：** `モダンな配色と大きなフォントでビジュアルデザインを改善`
- **プログレスバー：** `残り時間を表示するビジュアルプログレスバーを追加`

覚えておいてください：各機能の後にテストし、成功したらコミットし、失敗したら破棄。

## トラブルシューティング

- **「not a git repository」エラー** - timerフォルダにいることを確認（`cd ~/Documents/timer`）
- **FinderでTimer.htmlが見つからない** - ファイルは`/Users/YOUR_USERNAME/Documents/timer/timer.html`にあります
- **Gitコマンドが動作しない** - ステップ2を完了してGitをインストールしたことを確認
- **タイマーが動作しない** - ブラウザコンソールを開き（`Option (⌥) + Command (⌘) + I`を押して、**コンソール**タブをクリック）、赤いエラーメッセージをコピーしてClaudeに貼り付け

## Claudeに聞けること

- `どのファイルを変更した？` - コミットされていない変更を確認
- `差分を見せて` - 正確に何のコードが変更されたか確認
- `タイマーコードが何をするか説明して` - 実装を理解
- `experimentというブランチを作成` - リスクのある変更を安全に試す
- `前のコミットに戻って` - 最後のコミット以降のすべてを元に戻す

Claudeが自然言語ですべてのGit操作を処理—コマンドを覚える必要なし！

## ワークフロー概要

- **セットアップ：** Gitを一度インストール、身元を一度設定
- **開発：** Claudeがコードを書き、あなたがブラウザでテスト
- **バージョン管理：** ClaudeがシンプルなリクエストですべてのGit操作を処理
- **安全性：** いつでも悪い変更を破棄、任意の以前のコミットに戻れる
- **ローカル：** すべてがコンピュータ上に保存、アカウントやインターネット不要
