# WSLでClaude Code用にVS Codeをセットアップ

WindowsマシンにWSLとClaude Codeをインストールしました。次はコードを操作するためのビジュアルエディターが必要です。VS CodeはWindowsとLinux環境の橋渡しをし、統合ターミナルでClaude Codeを実行しながらファイルを視覚的に編集できます。VS CodeをLinux世界への窓口と考えてください。

## 主要な概念

- **VS Code** - Windows上で動作しますが、WSLに接続できるMicrosoftの無料コードエディター
- **WSL拡張機能** - VS CodeをLinux環境に接続し、Claude CodeなどのLinuxツールを実行できるようにします
- **統合ターミナル** - WSL（Linux）環境で実行されるVS Code内のターミナルパネル
- **/mnt/c/** - WSLがWindowsファイルにアクセスする方法（例：`/mnt/c/Users/...` = `C:\Users\...`）

## 必要なもの

- [WindowsへのClaude Codeのインストール](./Install_CLAUDE_Code_Win)を完了
- [VS Codeの基本](./VS_Code_Getting_Started)を完了
- 10〜15分

## ステップ1：プロジェクトフォルダを作成

- **エクスプローラー**を開く（タスクバーのフォルダアイコンをクリック）
- **ドキュメント**に移動
- 空白部分を右クリックし、**新規作成** > **フォルダー**を選択
- フォルダに`test_claude`と名前を付ける

## ステップ2：VS Codeを起動

- **Windowsスタートボタン**をクリック（画面左下）
- 検索ボックスに`Visual Studio Code`または`VS Code`と入力
- 検索結果に表示された**Visual Studio Code**をクリック
- VS Codeがウェルカムタブとともに開きます - このタブは閉じて構いません


## ステップ3：WSL拡張機能をインストール

- VS Codeで、左サイドバーの**拡張機能**アイコンをクリック（4つの四角形のように見えます）
- 検索ボックスに`WSL`と入力
- **WSL**（Microsoftによる）を見つけます（最初の結果のはずです）
- 青い**インストール**ボタンをクリック
- インストールが完了するまで待ちます（数秒）

## ステップ4：VS CodeをWSLに接続

- VS Codeの左下隅を見てください - 青または緑のアイコンがあります
- このアイコンをクリック（または`F1`を押して「WSL」と入力）
- メニューから**WSLへの接続**を選択
- VS Codeがリロードし、Ubuntuインストールに接続します
- 左下隅に**WSL: Ubuntu**と表示されるはずです

初めて接続する際、VS CodeはWSLに小さなサーバーをインストールします。約30秒かかります。

## ステップ5：VS Codeでフォルダを開く

- VS Code（WSLに接続したまま）で、メニューバーの**ファイル**をクリックし、**フォルダーを開く**をクリック
- 上部中央に**フォルダーを開く**ドロップダウンが表示されます
- 以下を入力してフォルダを見つけます：
  ```
  /mnt/c/Users/YOUR_USERNAME/Documents/test_claude
  ```
  `YOUR_USERNAME`をWindowsのユーザー名に置き換えてください（例：`John.Smith`）
- **OK**をクリック。VS Codeが`test_claude`フォルダでリロードされます
- 「このフォルダー内のファイルの作成者を信頼しますか？」と聞かれたら、**はい、作成者を信頼します**をクリック


## ステップ6：Claude Codeを起動

- VS Codeがリロードした後、新しいターミナルを開きます：メニューバーの**ターミナル**をクリックし、**新しいターミナル**をクリック
- ターミナルパネルで以下を入力：
  ```
  claude
  ```
- Claude Codeが起動します - おなじみのClaude Codeインターフェースが表示されます
- これでVS Codeでファイルを編集しながらClaude Codeを使用する準備が整いました

## ステップ7：ワークフローをテスト

- Claude Codeで以下を入力：
```
LLMがMarkdown形式を好んで使う理由を説明する短い記事を書いてください。article.mdとして保存してください
```
- Claude Codeがファイルを作成します - 左側のエクスプローラーパネルに`article.md`が表示されます
- エクスプローラーで`article.md`をクリックしてエディターで表示
- フォーマットされた記事をプレビューするには：`article.md`タブを右クリックし、**プレビューを開く**を選択
- 適切な見出し、箇条書き、フォーマットでMarkdownがレンダリングされます

## 後でVS CodeでClaudeを再開する

VS Codeを閉じた後、WSLプロジェクトに戻る方法：

- **オプションA：** VS Codeを開き、左下隅をクリックし、**WSLへの接続**を選択し、**ファイル** > **最近使用した項目を開く**をクリック。'/mnt/c/Users/YOUR_USERNAME/Documents/test_claude[WSL:Ubuntu 24.01]'を選択
- **オプションB：** WindowsからUbuntuアプリを開き、ターミナルでプロジェクトに移動し、`code .`と入力

VS Codeは最近のフォルダを記憶するので、「最近使用した項目を開く」のオプションAが通常最も速いです。

## 次のステップ

- 既存のコードベースについてClaude Codeに説明を求める：「このプロジェクトが何をするか説明して」
- 新機能の作成をClaude Codeに依頼：「リストの平均を計算する関数を追加して」
- バグ修正にClaude Codeを使用：「このコードがエラーを出すので、修正できますか？」
- インラインdiff付きのビジュアルインターフェース用にClaude Code VS Code拡張機能を試す（拡張機能で「Claude Code」を検索）
- 大きなファイルをより効率的に処理するには、WSLファイルシステム（/home/user/）にフォルダを作成します。Windowsからは、エクスプローラーのサイドバー下部にあるLinuxアイコンを探すか、エクスプローラーに`\\wsl.localhost\`と入力してください。

## トラブルシューティング

- **左下に「WSL: Ubuntu」が表示されない** - WSLが正しくインストールされていることを確認してください。まずUbuntuターミナルを開いて動作を確認してみてください
- **ターミナルにLinuxではなくPowerShellが表示される** - ターミナルパネルの**+**の横にあるドロップダウン矢印をクリックし、**Ubuntu (WSL)**を選択
- **`claude`コマンドが見つからない** - `claude --version`を実行してClaude Codeがインストールされているか確認。インストールされていない場合は、まずWSLインストールチュートリアルに従ってください

## ワークフロー概要

- **VS Code**はWindows上で実行され、ビジュアルエディターインターフェースを提供
- **WSL拡張機能**はVS CodeをUbuntuに接続し、Linuxツールを実行可能に
- **統合ターミナル**はWSL内でClaude Codeを実行
- ファイルはWindows（ドキュメントフォルダ）に保存されたまま - WSLは`/mnt/c/`経由でアクセス
- エディターでファイルを編集し、ターミナルでClaude Codeとチャット - 両方の利点を活用
